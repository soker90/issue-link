## Qué es

I-Licitaciones es una plataforma web, creada por abrahampo1, que centraliza, analiza y monitoriza las licitaciones del sector público en España. Está construida con Laravel 12 y plantillas Blade, y expone tanto una web pública (ilicitaciones.com) como el código fuente en GitHub. Importa los datos de contratación mediante comandos de consola y los presenta con un panel de métricas, rankings y buscadores.

## Para qué sirve

- Consultar licitaciones públicas con filtros y ver su historial.
- Analizar qué organismos públicos concentran más volumen de contratación.
- Revisar las empresas adjudicatarias ordenadas por importe de contrato.
- Explorar tendencias y métricas agregadas del gasto público.

## Cuándo usarlo

- Cuando necesitas estudiar la contratación pública española sin montar tu propio sistema de ingesta de datos.
- Es útil para empresas que rastrean adjudicaciones, investigadores y periodismo de datos.
- Si quieres montar tu propia instancia, puedes clonar el repositorio y cargar los datos con los comandos `artisan` incluidos.

## Ejemplo

Flujo de uso de la web pública:

1. Entra en la sección "Organismos" para ver las administraciones ordenadas por volumen de contratación.
2. Pasa a "Empresas" para consultar los adjudicatarios principales y el importe de sus contratos.
3. Usa "Análisis" y "Últimas licitaciones" para ver métricas agregadas y los procesos más recientes.

Para desplegar tu propia instancia desde el repositorio:

```
git clone https://github.com/abrahampo1/ilicitaciones
cd ilicitaciones
composer install
npm install && npm run build
cp .env.example .env
php artisan key:generate
php artisan migrate
php artisan app:importar-categorias
php artisan app:importar-licitaciones
```

## Puntos clave

- Web pública gratuita con datos de cientos de miles de licitaciones, organismos y empresas.
- Código abierto en GitHub, basado en Laravel 12 (PHP 8.2+), Blade y Vite.
- Importación automatizada de datos mediante comandos de consola de Artisan.

## Ten en cuenta

- El repositorio no declara una licencia explícita, así que revisa los términos con el autor antes de reutilizar el código en un proyecto propio.
- No se documenta de forma clara la fuente de datos exacta que alimenta la importación, algo a verificar si vas a autoalojarlo.
- Para una instancia propia necesitas un entorno PHP completo (Composer, Node/NPM y MySQL o SQLite).
