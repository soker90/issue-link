## Qué es

Astro Portfolio Template es una plantilla open source creada por Maxence Wolff (MaeWolff) para montar un portfolio personal con una sección de blog integrada. Está construida con Astro como generador de sitios estáticos, Tailwind CSS para los estilos y TypeScript. El contenido del portfolio se configura mediante archivos de datos y los artículos del blog se escriben en Markdown.

## Para qué sirve

- Publicar un portfolio personal de desarrollador o creativo con presentación, proyectos y enlaces sociales.
- Mantener un blog escribiendo artículos directamente en Markdown.
- Editar presentación, proyectos, redes y colores desde archivos de configuración, sin tocar la estructura.
- Aprovechar la generación estática de Astro para un sitio rápido y fácil de desplegar.

## Cuándo usarlo

Es buena opción cuando quieres una web personal sencilla y con buen rendimiento sin invertir tiempo en montar la base. Pesa poco y prioriza Markdown y configuración por datos, así que encaja bien para perfiles que solo necesitan portfolio más blog. Si buscas algo más completo (i18n, MDX, RSS, colecciones tipadas), plantillas como AstroPaper o Astrowind cubren más casos.

## Ejemplo

Para clonar la plantilla y arrancarla en local:

```bash
git clone https://github.com/MaeWolff/astro-portfolio-template.git
cd astro-portfolio-template
npm install
npm run dev
```

Después se personalizan los datos (presentación, proyectos, redes, colores) en la carpeta de configuración y se añaden los artículos como archivos Markdown en el directorio de posts. Para publicar:

```bash
npm run build
npm run preview
```

## Puntos clave

- Portfolio y blog en una sola plantilla, con modo oscuro y tema de color configurable.
- Configuración por archivos de datos: cambias contenido sin tocar componentes.
- SEO ya resuelto con astro-seo, gestión de fuentes con astro-font y generación de robots.txt.
- Stack moderno con Astro 4 y Tailwind CSS; licencia MIT.

## Ten en cuenta

El proyecto sigue activo y con mantenimiento, bajo licencia MIT. No incluye soporte para MDX, RSS ni sitemap de serie, así que si los necesitas tendrás que añadirlos. Al ser una plantilla pensada para uso personal, conviene revisar y adaptar todos los datos de ejemplo antes de publicar para que el sitio tenga identidad propia.
