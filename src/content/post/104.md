## Qué es

vite-plugin-css-export es un plugin para Vite, creado por shixuanhong, que permite leer desde JavaScript variables definidas en tus archivos CSS (o en preprocesadores como Sass, Less y Stylus). Usa una pseudoclase `:export`: las propiedades declaradas dentro de ese bloque quedan accesibles como un objeto al importar el archivo de estilos con el sufijo `?export`. Es open source con licencia MIT.

## Para qué sirve

- Tener una única fuente de verdad para tokens de diseño (colores, tipografías, breakpoints) y consumirlos tanto en CSS como en JS.
- Pasar valores de tus estilos a librerías JavaScript que necesitan datos (por ejemplo, colores para una gráfica o una animación).
- Reutilizar variables de Sass/Less en lógica de cliente sin reescribirlas a mano en JavaScript.
- Aprovechar reglas anidadas de los preprocesadores y transformar nombres (kebab-case a camelCase) al exportarlos.

## Cuándo usarlo

Es útil en proyectos con Vite donde el diseño vive en CSS/Sass y necesitas esos mismos valores en JavaScript, evitando que se desincronicen al duplicarlos. Si ya defines tus tokens directamente en JS (por ejemplo con la config de Tailwind o un objeto de tema) puede que no lo necesites; este plugin brilla cuando la "fuente de verdad" está en los estilos.

## Ejemplo

Instalación:

```bash
npm install vite-plugin-css-export -D
```

Configurar el plugin en Vite:

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import ViteCSSExportPlugin from 'vite-plugin-css-export';

export default defineConfig({
  plugins: [ViteCSSExportPlugin()],
});
```

Exportar variables desde CSS y leerlas en JS:

```css
/* example.css */
:export {
  fontColor: #333;
  fontSize: 14px;
}
```

```typescript
// main.ts
import cssResult from './example.css?export';
console.log(cssResult); // { fontColor: "#333", fontSize: "14px" }
```

## Puntos clave

- Comparte variables entre estilos y JavaScript usando el bloque `:export`.
- Compatible con CSS, Sass, Less y Stylus, incluyendo reglas anidadas y CSS Modules.
- Opciones para transformar nombres de propiedades (kebab-case → camelCase).
- Ligero, sin dependencias de runtime: actúa en tiempo de compilación de Vite.
- Open source (MIT) y mantenido activamente.

## Ten en cuenta

- Es específico de Vite; no sirve para proyectos con Webpack u otros bundlers.
- Las variables salen como cadenas de texto (`"14px"`), no como números, así que ojo al usarlas en cálculos.
- Si tu sistema de tokens ya está en JavaScript, añadir este plugin puede ser una capa innecesaria.
