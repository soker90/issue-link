## Qué es

Vite es la herramienta de frontend que ha reemplazado a webpack como estándar del ecosistema. En desarrollo sirve los módulos ES nativos del navegador sin pasar por un bundle, lo que hace que el servidor arranque en milisegundos. Para producción usa Rollup con tree-shaking agresivo.

## Para qué sirve

- Arrancar un proyecto React, Vue, Svelte, Lit o Vanilla en segundos.
- Tener HMR (Hot Module Replacement) instantáneo incluso en proyectos grandes.
- Generar builds de producción optimizados con code splitting automático.
- Extender la configuración mediante plugins del ecosistema Vite/Rollup.

## Cuándo usarlo

Prácticamente siempre para proyectos nuevos. Es el punto de partida recomendado por React, Vue y Svelte. Si vienes de Create React App o webpack y el arranque en local empieza a dolerte, Vite es la solución directa.

## Puntos clave

- Arranque del servidor de desarrollo casi instantáneo, sin importar el tamaño del proyecto.
- HMR preserva el estado del componente entre guardados.
- Configuración mínima por defecto, extensible con plugins de Rollup.
- TypeScript, JSX, CSS modules y assets estáticos soportados sin configuración extra.

## Ten en cuenta

La build de producción usa Rollup, no el mismo motor que el dev server. En proyectos muy grandes o con dependencias CommonJS complejas puede haber diferencias entre desarrollo y producción que hay que tener en cuenta.
