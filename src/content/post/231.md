## Qué es

SendScriptWhatsApp es un repositorio de código abierto divertido e informal que contiene scripts de automatización en JavaScript. Su propósito más famoso y extendido es permitir el envío del guion completo de la película _Shrek_ (o de _Bee Movie_) línea por línea y de forma consecutiva a un chat o grupo de WhatsApp, utilizándose habitualmente como una broma o spam consentido entre amigos.

## Para qué sirve

- Copiar y ejecutar scripts directamente en la consola de desarrollador del navegador (`F12`) mientras se tiene abierta la pestaña de WhatsApp Web.
- Automatizar el proceso de simular la escritura y el clic en el botón de envío de WhatsApp Web de forma ultra rápida.
- Enviar cualquier texto largo dividido en fragmentos individuales de manera desatendida y automática.
- Servir como ejemplo básico de scripting de consola y manipulación simple del DOM sobre aplicaciones de chat web.

## Cuándo usarlo

Es un recurso humorístico para realizar bromas inofensivas entre amigos enviándoles guiones completos de películas de forma masiva. También sirve como aprendizaje práctico sobre cómo inyectar scripts en la consola del navegador. Para automatizaciones comerciales serias y seguras de WhatsApp, se deben utilizar herramientas oficiales basadas en la API de WhatsApp Business.

## Puntos clave

- Cero dependencias: no requiere instalar Node.js ni configurar frameworks pesados; se ejecuta directamente en la consola del navegador.
- Los guiones de las películas están pre-cargados en formato de array dentro del propio archivo de script.
- Utiliza métodos del navegador para imitar las pulsaciones de teclas e interactuar con los elementos del DOM de WhatsApp Web.
- Es un repositorio muy clonado ("forked") por la comunidad para añadir nuevos guiones y adaptarlo a los cambios en la interfaz de WhatsApp.

## Ten en cuenta

> [!WARNING]
> El uso continuado de scripts automáticos de envío puede violar las condiciones de servicio de WhatsApp y provocar la suspensión o el bloqueo temporal de tu número de teléfono por actividad sospechosa de spam. Además, los navegadores modernos pueden bloquear el pegado de código directo en la consola por seguridad (requiriendo escribir primero `allow pasting`).
