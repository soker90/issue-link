## Qué es

Best-README-Template es una plantilla de README en Markdown creada por Othneil Drew. Recoge la estructura habitual de un buen archivo README (presentación del proyecto, instalación, uso, hoja de ruta y contribución) en un único fichero listo para copiar y adaptar. Se distribuye bajo licencia Unlicense (dominio público) y es uno de los repositorios de documentación más populares de GitHub.

## Para qué sirve

- Documentar un proyecto nuevo partiendo de una estructura ya probada.
- Mantener un formato de README coherente entre varios proyectos o equipos.
- Ahorrar tiempo evitando reescribir las mismas secciones una y otra vez.
- Aprovechar plantillas auxiliares para reportes de bugs y peticiones de funcionalidad.

## Cuándo usarlo

Es útil cuando arrancas un proyecto y quieres una documentación clara sin pensar en la estructura desde cero, o cuando gestionas varios repositorios y buscas un estilo uniforme. Incluye dos variantes: la versión completa con ejemplos e insignias de tecnologías, y un `BLANK_README.md` minimalista para empezar rápido.

## Ejemplo

Flujo básico de uso:

1. Descarga el archivo README.md (o BLANK_README.md) desde el repositorio.
2. Colócalo en la raíz de tu proyecto.
3. Sustituye los datos de ejemplo: nombre del proyecto, descripción, logo, enlaces y secciones.
4. Completa o elimina las secciones que no apliques (Getting Started, Usage, Roadmap, Contributing).
5. Ajusta las insignias de tecnologías y los enlaces de navegación "back to top".

## Puntos clave

- Estructura completa: About, Getting Started, Usage, Roadmap, Contributing, License y Contact.
- Incluye insignias preparadas para tecnologías comunes (React, Vue, Angular, etc.).
- Aporta plantillas de issues para reportes de bugs y peticiones de funcionalidad.
- Licencia Unlicense (dominio público), sin restricciones de uso.

## Ten en cuenta

Es solo una plantilla de documentación: no genera nada de forma automática, hay que rellenarla y personalizarla a mano. Conviene eliminar las secciones de ejemplo que no uses para que el README no quede recargado. Si prefieres algo más mínimo, usa el `BLANK_README.md` en lugar de la versión completa.
