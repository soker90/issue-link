## Qué es

Flowbite Astro Admin Dashboard es una plantilla open source de panel de administración creada por Themesberg, los autores de Flowbite. Está construida con Astro, Tailwind CSS y la librería de componentes Flowbite, con TypeScript en modo estricto. Se distribuye bajo licencia MIT e incluye más de quince páginas de ejemplo listas para usar.

## Para qué sirve

- Montar el back-office o panel interno de una aplicación sin partir de cero.
- Reaprovechar páginas ya hechas: dashboard, gestión de productos y usuarios (CRUD), ajustes y perfil.
- Disponer de pantallas de autenticación (login, registro, recuperación de contraseña) y de error (404, 500).
- Aprender cómo integrar Flowbite y ApexCharts dentro de un proyecto Astro real.

## Cuándo usarlo

Encaja cuando necesitas un panel de administración y ya trabajas con Astro y Tailwind, y prefieres una base con componentes y gráficas resueltos antes que ensamblarlo todo a mano. Si tu proyecto no usa Astro, Themesberg mantiene variantes equivalentes en HTML/Tailwind puro y en otros frameworks; valóralas para no atarte a Astro sin necesidad.

## Ejemplo

Clonar la plantilla y arrancar el entorno de desarrollo:

    git clone https://github.com/themesberg/flowbite-astro-admin-dashboard.git
    cd flowbite-astro-admin-dashboard
    npm install
    npm run dev

A partir de ahí se editan las páginas dentro de src/pages y los componentes en src/components, y se ajusta la configuración de Tailwind y Flowbite a la identidad del proyecto.

## Puntos clave

- Más de quince páginas de ejemplo: dashboard, CRUD, autenticación, ajustes y errores.
- Componentes de Flowbite ya integrados: tablas, modales, drawers, tooltips y date pickers.
- Gráficas con ApexCharts y soporte de modo oscuro incluido.
- TypeScript en modo estricto, ESLint, Prettier y despliegue a GitHub Pages vía GitHub Actions.

## Ten en cuenta

Es un punto de partida, no un producto terminado: hay que personalizar estilos, datos y lógica para darle identidad propia. Usa Tailwind CSS v3 y Flowbite, así que conviene revisar la compatibilidad de versiones antes de actualizar dependencias. El ritmo de commits del repositorio es bajo, de modo que valida que las versiones de Astro y Flowbite encajan con las de tu proyecto.
