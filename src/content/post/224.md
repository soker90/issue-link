## Qué es

readme-md-generator es una herramienta de línea de comandos (CLI) que simplifica enormemente la creación de archivos `README.md` profesionales para cualquier proyecto. En lugar de comenzar a escribir desde una plantilla vacía, este script analiza el entorno local del repositorio (como archivos `package.json` y el historial de git) para completar automáticamente la mayor parte de la información técnica básica y presentar preguntas interactivas para el resto de los detalles.

## Para qué sirve

- Generar en pocos segundos un archivo README estructurado que contiene secciones de instalación, uso, tests, autoría, contribuciones y licencias.
- Autodetectar e incluir badges dinámicas de npm, de versión de node, y de estado del build de GitHub Actions.
- Configurar la estructura de documentación de manera interactiva directamente en la terminal sin necesidad de escribir código Markdown a mano.
- Crear y reutilizar plantillas personalizadas (usando EJS) para asegurar un formato consistente en todos los proyectos de una organización.

## Cuándo usarlo

Es ideal al iniciar nuevos proyectos o al limpiar repositorios antiguos que carecen de una buena documentación. Su uso previene el tedio de crear archivos de documentación desde cero, asegurando que los metadatos más importantes (versiones, links del repositorio, comandos de instalación) sean siempre correctos y estén actualizados. Como alternativas existen generadores basados en web, pero este CLI destaca por poder ejecutarse localmente de forma automatizada.

## Puntos clave

- Altamente intuitivo, hace preguntas de confirmación sencillas ("¿cuál es la licencia?", "¿cuál es el comando para test?").
- Lee metadatos de git para extraer el nombre del autor, el email y el usuario de GitHub para las secciones de contacto.
- Soporta la adición instantánea de instrucciones para instalar dependencias con múltiples gestores (npm, yarn).
- Genera un diseño visual moderno con badges coloridas e iconos de formato atractivos y estandarizados.

## Ten en cuenta

Dado que depende de metadatos locales, si tu proyecto no tiene un archivo de manifiesto (como `package.json`) o no está inicializado con git, el generador no podrá autocompletar la información básica y tendrás que escribirla manualmente en la terminal.
