## Qué es

Dokto es una plantilla gratuita y de código abierto para crear la landing page de una clínica, consulta o servicio médico, desarrollada por el estudio cssninjaStudio. Está construida con Astro como framework base, Alpine.js para la interactividad ligera y Tailwind CSS para los estilos. Su enfoque es servir como punto de partida pulido y accesible para presentar online un negocio del sector salud.

## Para qué sirve

- Montar rápidamente la web de una clínica, consultorio o profesional médico sin partir de cero.
- Aprovechar una maquetación ya resuelta con secciones típicas de una landing page.
- Disponer de una base accesible (WCAG) sin tener que configurarla desde el principio.
- Aprender cómo se combinan Astro, Alpine y Tailwind en un proyecto real y bien estructurado.

## Cuándo usarlo

Es una buena opción cuando necesitas publicar la página de presentación de un negocio sanitario y prefieres personalizar una plantilla cuidada en lugar de diseñarla desde cero. Como es muy específica del sector médico, si tu proyecto es de otro tipo quizá encajen mejor otros starters del mismo estudio, como Fresh.

## Ejemplo

Clonar el repositorio e instalar dependencias para arrancar el entorno de desarrollo local:

```
git clone https://github.com/cssninjaStudio/dokto.git
cd dokto
npm install
npm run dev
```

A partir de ahí se editan los contenidos y secciones de la landing y se genera la versión estática con npm run build.

## Puntos clave

- Stack moderno: Astro para el renderizado, Alpine.js para interactividad y Tailwind CSS para los estilos.
- Pensada para accesibilidad, con foco en cumplir pautas WCAG (semántica, landmarks, navegación por teclado).
- Gratuita, con licencia MIT, lo que permite usarla y modificarla libremente.

## Ten en cuenta

La plantilla está orientada específicamente al sector salud (citas y servicios médicos), por lo que adaptarla a otro tipo de negocio puede requerir bastante trabajo de rediseño. El proyecto se mantiene activo, aunque con un ritmo de actualización moderado; conviene revisar las versiones de Astro y demás dependencias antes de partir de ella.
