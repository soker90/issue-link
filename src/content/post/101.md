## Qué es

shadcn/ui es una colección de componentes de interfaz reutilizables, creada por shadcn (en Vercel). Su enfoque es distinto al de una librería tradicional: en lugar de instalar un paquete y depender de él, **copias el código de cada componente a tu propio proyecto** mediante una CLI, y a partir de ahí es tuyo para editar. Está construido sobre Radix UI (accesibilidad y comportamiento) y Tailwind CSS (estilos), pensado principalmente para React (Next.js, Vite, Remix, Astro…). Es open source con licencia MIT.

## Para qué sirve

- Añadir componentes accesibles y bien diseñados (botones, diálogos, formularios, tablas, menús…) sin construirlos desde cero.
- Tener control total del código: como vive en tu repo, puedes adaptar estilos y comportamiento sin pelearte con la API de una librería externa.
- Mantener una base coherente de diseño usando tokens de Tailwind y variables de tema (incluido modo oscuro).
- Evitar el "lock-in": no hay dependencia de versión que te bloquee, porque el código es tuyo.

## Cuándo usarlo

Es ideal cuando trabajas con React + Tailwind y quieres componentes de calidad que puedas personalizar a fondo. Frente a librerías "todo en uno" como Material UI o Mantine (instalas y consumes), shadcn/ui te da el código fuente para que sea tu sistema de diseño. Si prefieres no gestionar el código de los componentes en tu repo o no usas Tailwind, una librería empaquetada puede encajar mejor.

## Ejemplo

Inicializar shadcn/ui en un proyecto y añadir componentes con la CLI:

```bash
# Configura shadcn/ui en tu proyecto (Tailwind, alias, tema...)
npx shadcn@latest init

# Añade los componentes que necesites (copian su código a tu repo)
npx shadcn@latest add button dialog
```

```tsx
import { Button } from "@/components/ui/button";

export function Ejemplo() {
  return <Button variant="outline">Aceptar</Button>;
}
```

## Puntos clave

- Modelo "copia el código": los componentes viven en tu proyecto, no en `node_modules`.
- Accesibilidad de serie gracias a Radix UI; estilos con Tailwind CSS y soporte de temas y modo oscuro.
- CLI (`npx shadcn@latest`) para inicializar y añadir componentes uno a uno.
- Agnóstico de framework dentro del ecosistema React/JSX (Next.js, Vite, Remix, Astro…).
- Muy popular y activo, con licencia MIT.

## Ten en cuenta

- Requiere Tailwind CSS; si tu proyecto no lo usa, no es la opción natural.
- Al copiar el código a tu repo, las mejoras posteriores del proyecto no llegan solas: actualizar un componente implica volver a traerlo o aplicar los cambios a mano.
- No es una librería versionada: ganas control y personalización a cambio de mantener tú ese código.
- El paquete de la CLI pasó de llamarse `shadcn-ui` a `shadcn`; usa `npx shadcn@latest`.
