## Qué es

Maybe es una aplicación de finanzas personales open source para gestionar cuentas, transacciones, presupuestos y seguimiento del patrimonio. Nació en 2021 como un producto comercial de la empresa Maybe Finance; tras cerrar el negocio, el código se liberó como proyecto open source. Está construido con Ruby on Rails, Hotwire/Turbo y Stimulus, sobre PostgreSQL, y se publica bajo licencia AGPLv3.

## Para qué sirve

- Centralizar tus cuentas y transacciones financieras en una instancia propia.
- Hacer seguimiento del patrimonio neto y de inversiones a lo largo del tiempo.
- Crear presupuestos y categorizar gastos sin depender de servicios externos.
- Estudiar una app de finanzas real hecha con Rails y Hotwire, o hacer fork para adaptarla.

## Cuándo usarlo

Está pensado para quien quiere controlar sus finanzas personales en un entorno self-hosted, con sus datos en su propio servidor. Es buena opción si te manejas con Docker y valoras la privacidad frente a apps SaaS de finanzas. Si buscas un proyecto con mantenimiento activo, considera el fork comunitario Sure, que continúa el desarrollo, u otras alternativas como Firefly III o Actual Budget.

## Ejemplo

Despliegue local básico con Docker Compose:

```
git clone https://github.com/maybe-finance/maybe.git
cd maybe
cp .env.example .env
docker compose up -d
```

La app queda disponible en el navegador (por defecto en http://localhost:3000), donde creas tu usuario y empiezas a añadir cuentas y transacciones.

## Puntos clave

- Self-hosted: tus datos financieros se quedan en tu infraestructura.
- Stack Ruby on Rails con Hotwire/Turbo y Stimulus, base de datos PostgreSQL.
- Despliegue sencillo mediante Docker Compose.
- Licencia AGPLv3, con código abierto para inspección, fork y contribución.

## Ten en cuenta

El repositorio está archivado y ya no recibe mantenimiento oficial: la empresa Maybe Finance cerró y el proyecto quedó como open source self-hosted, con la v0.6.0 como última versión. No esperes nuevas funciones ni parches de seguridad del equipo original. Si quieres una versión viva, existe el fork comunitario Sure, que mantiene el proyecto (los forks deben respetar la AGPLv3 y no usar la marca "Maybe"). Al gestionar datos financieros sensibles, asegura bien tu despliegue.
