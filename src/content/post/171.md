## Qué es

Astro Ink es una plantilla (theme) de blog personal para el framework Astro, creada por one-aalam y publicada bajo licencia MIT. Es minimalista y está centrada en el rendimiento y la escritura en Markdown. Combina Astro 2.x con Tailwind CSS para los estilos y Svelte para los componentes interactivos, e incluye TypeScript de serie.

## Para qué sirve

- Lanzar rápidamente un blog personal o de contenidos sin construir el theme desde cero.
- Escribir entradas en Markdown/Markdoc con paginación, tags y tarjetas de autor ya resueltas.
- Personalizar la apariencia gracias a varios esquemas de color y modo oscuro incluidos.
- Usarlo como base de aprendizaje para entender un proyecto Astro real y bien estructurado.

## Cuándo usarlo

Encaja cuando quieres un blog ligero y rápido en Astro y prefieres adaptar una plantilla mantenida antes que partir de un proyecto en blanco. Si solo necesitas el starter oficial mínimo, las plantillas base de Astro pueden bastar; si buscas algo más completo (búsqueda en cliente, temas, CMS), Astro Ink aporta esas piezas ya integradas.

## Ejemplo

Para empezar a partir de la plantilla con la CLI de Astro:

    npm create astro@latest -- --template one-aalam/astro-ink
    cd astro-ink
    npm install
    npm run dev

También puedes clonar el repositorio directamente y arrancar el servidor de desarrollo en local antes de personalizar el contenido y los estilos.

## Puntos clave

- Stack moderno: Astro, Tailwind CSS y componentes Svelte para las partes interactivas.
- Búsqueda en cliente, modo oscuro y varios esquemas de color listos para usar.
- Extras como integración con Netlify CMS, contador de vistas y publicación programada vía GitHub Actions.

## Ten en cuenta

Está construido sobre Astro 2.x, una versión ya antigua del framework, así que conviene revisar la compatibilidad y planificar una posible actualización a versiones recientes de Astro. Algunas funciones avanzadas (contador de vistas con Redis, CMS) requieren configuración adicional. Es una plantilla de punto de partida: tendrás que adaptar contenido, estilos y dependencias a tu proyecto.
