## Qué es

ts-pattern es una librería de *pattern matching* (coincidencia de patrones) para TypeScript y JavaScript, creada por Gabriel Vergnaud. Permite expresar lógica condicional compleja como una única expresión declarativa en lugar de anidar `if/else` o `switch`, con inferencia de tipos inteligente y comprobación de exhaustividad en tiempo de compilación. Pesa alrededor de 2 kB y se distribuye con licencia MIT.

## Para qué sirve

- Sustituir condicionales anidados por una expresión `match().with(...)` legible y tipada.
- Discriminar uniones de TypeScript (por ejemplo, estados `loading | success | error`) con narrowing automático del tipo en cada rama.
- Garantizar en compilación que has cubierto todos los casos posibles con `.exhaustive()`.
- Hacer coincidir estructuras complejas: objetos, arrays, tuplas, sets, maps y primitivos.
- Extraer valores de dentro de un patrón con `P.select()` sin código extra.

## Cuándo usarlo

Encaja cuando tienes lógica basada en uniones discriminadas o en la forma de los datos y quieres que el compilador te avise si olvidas un caso. Es especialmente útil en reductores de estado, máquinas de estado o parsers. Si solo tienes dos o tres ramas simples, un `switch` normal puede bastar; ts-pattern brilla cuando la exhaustividad y el narrowing aportan seguridad real.

## Ejemplo

```bash
npm install ts-pattern
```

```typescript
import { match, P } from 'ts-pattern';

type Resultado =
  | { tipo: 'cargando' }
  | { tipo: 'ok'; datos: string[] }
  | { tipo: 'error'; mensaje: string };

const render = (r: Resultado): string =>
  match(r)
    .with({ tipo: 'cargando' }, () => 'Cargando...')
    .with({ tipo: 'ok' }, ({ datos }) => `${datos.length} elementos`)
    .with({ tipo: 'error' }, ({ mensaje }) => `Error: ${mensaje}`)
    .exhaustive(); // error de compilación si falta cubrir un caso
```

## Puntos clave

- Inferencia de tipos de primera clase: cada rama recibe el tipo ya estrechado (narrowed).
- `.exhaustive()` convierte los casos no cubiertos en errores de compilación.
- Patrones expresivos: comodines (`P._`), por tipo (`P.string`, `P.number`), guardas con `.when()` y extracción con `P.select()`.
- Muy ligera (~2 kB) y sin dependencias.
- Funciona tanto en TypeScript como en JavaScript; licencia MIT y proyecto activo.

## Ten en cuenta

- El máximo valor lo aporta en TypeScript: en JavaScript puro pierdes la comprobación de exhaustividad y el narrowing.
- Añade una dependencia y una pequeña curva de aprendizaje frente a un `switch` nativo; para condicionales triviales puede ser innecesaria.
- La documentación vive en el README del repositorio (no hay sitio de documentación aparte).
