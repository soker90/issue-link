## Qué es

Zed es un editor de código de escritorio escrito desde cero en Rust por Zed Industries, el equipo detrás de Atom y Tree-sitter. Aprovecha múltiples núcleos de CPU y la GPU mediante su propio framework gráfico, GPUI, para conseguir una interfaz muy fluida. Es open source con licencia GPL-3.0 (con componentes Apache-2.0) y está en desarrollo muy activo.

## Para qué sirve

- Editar y navegar código con baja latencia incluso en proyectos grandes.
- Programar en pareja o en equipo con edición colaborativa en tiempo real.
- Usar asistencia de IA: edición agéntica, asistente en línea y predicción de ediciones.
- Trabajar con soporte nativo de Git, depurador (DAP), multibuffer, vista de esquema y REPL.

## Cuándo usarlo

- Cuando el rendimiento y la respuesta del editor son una prioridad y VS Code te resulta pesado.
- Cuando quieres colaborar en directo sobre el mismo proyecto sin herramientas externas.
- Cuando buscas integrar agentes de IA en el flujo de edición, con la opción de usar tus propias claves de API.
- Como alternativa a VS Code, Sublime Text o JetBrains; menos extensiones que VS Code, pero más velocidad.

## Ejemplo

Instalación en macOS o Linux desde la terminal:

```
# macOS (Homebrew)
brew install zed

# Linux (script oficial)
curl -f https://zed.dev/install.sh | sh
```

También hay descargas directas para macOS, Linux y Windows en la web oficial. Para abrir un proyecto desde la terminal una vez instalado:

```
zed .
```

## Puntos clave

- Núcleo del editor escrito íntegramente en Rust sobre el framework GPUI.
- Colaboración multiusuario en tiempo real integrada de fábrica.
- IA flexible: edición agéntica y predicción, sin atar a un único modelo o proveedor.
- Soporte nativo de Git, depurador DAP, edición modal estilo Vim y resaltado con Tree-sitter.

## Ten en cuenta

- El editor es gratuito para siempre; las funciones de IA gestionadas por Zed son de pago (plan Pro a 10 $/mes y Business a 30 $/usuario/mes), aunque puedes usar tus propias claves de API o desactivar la IA por completo.
- Aún no existe versión web; solo aplicación de escritorio para macOS, Linux y Windows.
- El ecosistema de extensiones es más pequeño que el de VS Code, así que conviene comprobar que existe soporte para tu lenguaje o herramienta.
