## Qué es

Magisk es una suite de personalización para Android (6.0 o superior) creada por John Wu (topjohnwu). Su rasgo distintivo es el enfoque "systemless": aplica root y modificaciones sin escribir en la partición del sistema, de modo que los cambios son fáciles de revertir y pueden ocultarse a ciertas apps. Está escrito principalmente en Kotlin, Rust y C++, y se distribuye como software libre bajo licencia GPL-3.0.

## Para qué sirve

- Obtener acceso root mediante MagiskSU para conceder permisos elevados a aplicaciones concretas.
- Instalar módulos (Magisk Modules) que modifican particiones de solo lectura sin tocarlas físicamente.
- Desempaquetar y reempaquetar imágenes de arranque de Android con MagiskBoot.
- Ejecutar código dentro de cualquier proceso de aplicación a través de Zygisk.

## Cuándo usarlo

- Cuando quieres rootear un dispositivo Android conservando la posibilidad de revertir los cambios y de recibir actualizaciones OTA.
- Cuando necesitas aplicar parches del sistema (módulos) de forma aislada y reversible, sin reescribir `/system`.
- Como alternativa moderna a métodos de root que sí modifican el sistema; existen también proyectos como KernelSU o APatch para enfoques basados en kernel.

## Ejemplo

Flujo básico para parchear una imagen de arranque y obtener root:

```
1. Desbloquea el bootloader del dispositivo (borra los datos).
2. Extrae el boot.img de la ROM/firmware oficial de tu dispositivo.
3. Instala la app Magisk (Magisk-vXX.X.apk) desde las releases de GitHub.
4. En la app: Instalar -> Seleccionar y parchear un archivo -> elige boot.img.
   Magisk genera magisk_patched-xxxxx.img.
5. Flashea esa imagen con fastboot:
   fastboot flash boot magisk_patched-xxxxx.img
6. Reinicia. La app Magisk gestionará el root y los módulos.
```

## Puntos clave

- Enfoque systemless: las modificaciones no alteran la partición del sistema y son reversibles.
- Módulos para extender el sistema y Zygisk para inyectar código en procesos de apps.
- MagiskBoot, una de las utilidades más completas para manipular imágenes boot de Android.
- Activamente mantenido por su autor original, con releases frecuentes (versión v30.7).

## Ten en cuenta

- Rootear requiere desbloquear el bootloader, lo que normalmente borra todos los datos del dispositivo y puede anular la garantía.
- Un dispositivo rooteado puede activar mecanismos como Play Integrity y dejar inutilizables apps de banca, pagos o DRM; la antigua función MagiskHide ya no forma parte del proyecto.
- Es una herramienta de bajo nivel: un parcheo incorrecto puede impedir el arranque (brick). Usa siempre la imagen boot oficial de tu dispositivo y haz copia de seguridad.
- Solo compatible con Android 6.0 y superior.
