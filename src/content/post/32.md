## Qué es

ToolHive Studio es la aplicación de escritorio open source de Stacklok para descubrir, instalar y ejecutar servidores MCP (Model Context Protocol). Está construida con Electron y TypeScript (interfaz en React con shadcn/ui) y forma parte de la plataforma ToolHive, que comparte runtime con su CLI y su operador de Kubernetes. Cada servidor MCP se lanza dentro de un contenedor aislado, lo que añade una capa de seguridad y reproducibilidad sobre el habitual proceso manual de configuración.

## Para qué sirve

- Descubrir e instalar servidores MCP desde un catálogo, sin editar archivos de configuración a mano.
- Ejecutar cada servidor MCP en un contenedor aislado con permisos y filtrado de red controlados.
- Conectar esos servidores a clientes y agentes de IA compatibles con MCP.
- Gestionar secretos e integraciones (incluida autenticación OAuth) desde una única interfaz gráfica.

## Cuándo usarlo

Es útil cuando trabajas con agentes o clientes de IA que consumen herramientas vía MCP y quieres montar y probar servidores sin pelearte con la configuración manual ni con la seguridad del entorno. Si prefieres flujos de terminal o automatización, la CLI de ToolHive cubre el mismo runtime; para despliegues de equipo o producción existe el operador de Kubernetes. ToolHive Studio es la opción de menor fricción para empezar en local.

## Ejemplo

Flujo típico de uso en la aplicación de escritorio:

1. Descarga e instala ToolHive Studio desde stacklok.com/download (Windows, macOS o Linux). Necesitas un runtime de contenedores como Docker.
2. Abre la app y explora el catálogo de servidores MCP disponibles.
3. Selecciona un servidor (por ejemplo, uno de acceso a sistema de ficheros o a una API) y pulsa instalar: ToolHive lo arranca en un contenedor aislado.
4. Ajusta permisos, variables y secretos del servidor desde la interfaz.
5. Conecta tu cliente o agente de IA compatible con MCP al servidor en ejecución y empieza a usar la herramienta.

## Puntos clave

- Ejecuta los servidores MCP en contenedores aislados, no como procesos sueltos en tu máquina.
- Permisos granulares y filtrado de acceso de red por servidor.
- Gestión integrada de secretos e integraciones OAuth empresariales.
- Comparte runtime con la CLI de ToolHive y el operador de Kubernetes, por lo que es coherente con flujos más avanzados.

## Ten en cuenta

- Requiere un runtime de contenedores (Docker u OCI) instalado en el sistema.
- Es una aplicación de escritorio en desarrollo activo y con releases frecuentes: revisa la versión y los cambios antes de depender de ella en flujos críticos.
- Licencia Apache 2.0. Para automatización o despliegues de servidor puede encajar mejor la CLI o el operador de Kubernetes que la app de escritorio.
