## Qué es

git-auto-commit-action es una GitHub Action que detecta cambios generados durante un workflow, crea un commit y lo sube al repositorio automaticamente.

## Para qué sirve

- Guardar archivos generados por CI.
- Actualizar documentacion, reports, datos o contenido estatico.
- Persistir resultados de scripts programados.
- Automatizar commits sin escribir comandos Git manuales en cada workflow.

## Cuándo usarlo

Usalo cuando un workflow modifica archivos del repositorio y quieres conservar esos cambios. Por ejemplo: generar markdown desde issues, actualizar precios, refrescar datasets, crear changelogs o guardar informes diarios.

## Puntos clave

- Expone si se detectaron cambios.
- Permite personalizar mensaje de commit, autor y archivos incluidos.
- Evita repetir boilerplate de `git add`, `git commit` y `git push`.

## Ejemplo tipico

```yaml
- name: Commit generated files
  uses: stefanzweifel/git-auto-commit-action@v4
  with:
    commit_message: 'Update generated content'
```

## Ten en cuenta

Hay que cuidar los bucles: un commit generado por el workflow puede volver a disparar otro workflow si no configuras bien ramas, paths o condiciones.
