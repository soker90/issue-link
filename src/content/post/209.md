## Qué es

rclone es una potente herramienta de línea de comandos (CLI) diseñada para sincronizar, copiar y gestionar archivos entre tu almacenamiento local y más de 70 proveedores de servicios de almacenamiento en la nube. Conocido frecuentemente como "el rsync para la nube", permite tratar carpetas de servicios remotos de la misma manera que directorios del sistema de archivos local.

## Para qué sirve

- Copiar y sincronizar directorios completos de forma unidireccional o bidireccional desde tu disco local hacia la nube, o entre múltiples proveedores de nube directamente.
- Montar almacenamientos en la nube (como Google Drive, Dropbox o AWS S3) como un sistema de archivos local utilizando FUSE para acceder a ellos como si fuesen discos físicos.
- Cifrar archivos en el lado del cliente (client-side encryption) antes de subirlos a la nube para garantizar la máxima privacidad de los backups.
- Automatizar migraciones de grandes volúmenes de datos mediante scripts programados, aprovechando características de transferencia paralela y control de ancho de banda.

## Cuándo usarlo

Es una de las mejores herramientas de infraestructura para administradores de sistemas, desarrolladores y entusiastas del self-hosting que necesitan gestionar copias de seguridad de forma automatizada o migrar datos entre nubes sin usar la CPU/red local como intermediario innecesario. Como alternativa directa destaca rsync para copias locales/SSH, pero rclone es el líder indiscutible para servicios en la nube.

## Puntos clave

- Admite transferencia multiproceso (multi-threaded), compresión al vuelo y algoritmos de control de integridad de archivos mediante hashes MD5 o SHA1.
- Incluye una interfaz web básica (GUI) experimental e interactiva opcional si prefieres no usar la terminal de comandos de forma exclusiva.
- Ejecución ultra ligera compilada en Go, compatible con múltiples plataformas como GNU/Linux, Windows, macOS, FreeBSD y arquitecturas ARM.
- Soporta comandos avanzados de filtrado para incluir o excluir tipos de archivos de forma muy flexible durante las copias.

## Ten en cuenta

Al ser una herramienta potente de línea de comandos, una mala configuración en los comandos de sincronización (como usar `rclone sync` en lugar de `rclone copy`) puede provocar la eliminación accidental de archivos en el destino si no se tiene cuidado. Se aconseja probar siempre con el parámetro `--dry-run` antes de ejecutar tareas reales.
