## Qué es

Bruno es un cliente de APIs de escritorio, open source y offline. Es una alternativa a Postman que guarda las colecciones como archivos en tu proyecto, lo que facilita versionarlas con Git.

## Para qué sirve

- Probar endpoints HTTP durante el desarrollo.
- Guardar colecciones de requests junto al codigo.
- Compartir pruebas de API sin depender de una nube propietaria.
- Documentar flujos de autenticacion, payloads y respuestas esperadas.

## Cuándo usarlo

Usalo si quieres un cliente de APIs moderno pero prefieres que tus colecciones vivan en el repositorio. Encaja muy bien en equipos que trabajan con Git y quieren revisar cambios en requests como cualquier otro archivo.

## Puntos clave

- Funciona offline.
- Las colecciones son archivos versionables.
- Buen sustituto si Postman se siente demasiado pesado o dependiente de cuenta/cloud.
- Tiene interfaz grafica, pero mantiene un enfoque developer-friendly.

## Ten en cuenta

Si tu equipo ya depende mucho del ecosistema colaborativo de Postman, migrar puede requerir ajustar procesos. Para uso local y repositorios versionados, Bruno es una opcion muy solida.
